# VR + AI 互動體驗重做設計規格

**日期**：2026-06-06
**狀態**：已確認方向，待實作（分階段）
**對象**：`technology.html` §01 VR demo（`technology.js` 的 `initVR`）

## 背景與目標

現有 VR demo 是「按 Start 看約 40 秒自動播放」的 cinematic loop，使用者只能被動觀看，缺少 VR 應有的「身歷其境、我在操控」感，使用者回饋「太無趣」。

目標：**全面重做成互動式體驗** —— 一個可探索、可操控的 3D 學習空間，使用者主導節奏。延續既有原則：純前端、無後端、無 AI API、程式即時渲染、不錄影。

## 確定的設計決策（來自使用者）

- **預設互動模式**：一進來就是可拖曳、可點擊的互動空間；另有按鈕可切換到「自動導覽」（保留現有 cinematic 當 fallback）。
- **對話形式**：預設劇本對話（點選項/點導師 → 導師說預寫台詞、出題、給回饋）。無真實 AI、無後端、無麥克風。
- **互動方式（全要）**：滑鼠拖曳環視場景、點擊場景物件觸發學習、跟 AI 導師（劇本）對話、可選場景/語言。
- **視覺升級（全要）**：更豐富 3D 場景細節、HUD 技術數據層、手部/控制器視覺、實拍背景更融入。
- **執行方式**：先寫完整 spec（本文件）→ 再寫實作計畫 → 分階段做（先互動骨架 → 再視覺升級），逐階段驗證。

## 互動設計

### 1. 拖曳環視（OrbitControls）
- 進入場景後，滑鼠拖曳 360° 環視 3D 空間。
- 限制俯仰角（避免看到場景穿幫/迷失），限制縮放範圍（滾輪小幅推近/拉遠）。
- 停止操作數秒後相機可緩慢自動回正或輕微飄移（保持畫面有生命感）。

### 2. 點擊場景物件觸發學習（核心互動）
- 場景關鍵物件（咖啡杯、菜單、航班看板、簡報螢幕、登機證等）帶**發光可點擊提示環**（hover 時加亮 + 游標變 pointer）。
- 用 Three.js `Raycaster` 偵測點擊。點下去：
  1. 相機平滑聚焦到該物件（lerp 過去）。
  2. 浮出該物件的多國語單字卡（英/中/日）+ 例句 + 發音回饋環（綠=準/黃=再試，純視覺）。
  3. AI 導師（能量球）轉向它、說一句相關引導台詞（劇本）。
  4. HUD 數據更新（當前注視物件名、發音評分動畫）。

### 3. AI 導師劇本對話
- 畫面一角固定一個導師對話框（HTML/CSS 疊層）。
- 導師主動引導（「試試點點看那杯咖啡」「很好，現在問問價格」）。
- 點物件後導師給回應，並提供 2-3 個**選項按鈕**（例：點完咖啡後出現「a flat white, please」「how much is it?」）。
- 點選項 → 推進對話 + 加分 + 導師下一句。純劇本分支（有限狀態，不是真 AI）。

### 4. 選場景 / 選語言
- HUD 上可切換**場景**（Café / Airport / Meeting）與**目標語言**（英 / 日 /（中））。
- 自由探索，不強制固定順序。切換場景時用既有的光帶轉場。

## 視覺升級

### A. 更豐富 3D 場景
- 增加物件密度（每場景更多低多邊形道具）。
- 加粒子系統（漂浮光點 / 塵埃 / 能量微粒）。
- 動態燈光（呼吸光暈、物件聚光、導師移動時的光影）。
- 物件 hover 發光、可點物件持續微脈動提示。

### B. HUD 技術數據層
- 中央準星 / 追蹤點（瞄準當前物件）。
- 發音評分條（點物件後動畫填充）。
- 語意分析小雷達 / 條狀圖（裝飾性，呼應實驗室數據美學）。
- 座標 / FPS / 當前注視物件名稱（mono 字體科技感）。
- 既有的 IMMERSION% / LATENCY 保留並擴充。

### C. 手部 / 控制器視覺
- 畫面下方一對程式渲染的 VR 控制器（低多邊形）或光線指標（raycaster 視覺化）。
- 跟滑鼠移動，點擊時射出光束打到物件（命中閃光）。
- 強化「這是 VR」的印象。

### D. 實拍背景更融入
- Codex 生成的背景圖（images/vr-{cafe,airport,meeting}.jpg）配合場景做色調統一。
- 加地面倒影或邊緣暈影，讓 3D 物件像真站在那空間裡。
- 背景與場景霧 / 燈光色一致。

## 技術方案

- **Three.js**（已有 r160 CDN）+ **OrbitControls**（同來源加一個 CDN script）+ **Raycaster**（Three.js 內建，點擊偵測）。
- 仍純前端、無後端、無 AI API、無麥克風。
- 場景資料參數化：擴充既有 `SCENES`，每個場景加一份**可互動物件清單**（每物件：3D 位置、幾何、多國語單字 / 例句、導師台詞、對話選項）。
- 對話系統：有限狀態機 + 劇本資料（每場景一份對話樹）。
- **保留「▶ 自動導覽」模式**：現有 cinematic 三幕邏輯保留為一個模式，互動模式為預設，按鈕可切換。
- `prefers-reduced-motion`：互動模式仍可用（拖曳是使用者主動），但停用自動飄移 / 粒子動畫，或退回靜態互動。
- 效能：只在 demo 啟動且可見時跑 rAF；OrbitControls 的 damping 用 rAF 驅動。

## 分階段實作（概念，細節進實作計畫）

1. **階段一 · 互動骨架**：OrbitControls 環視 + Raycaster 點擊物件 + 點擊聚焦 + 基本可點物件（沿用現有三場景）。模式切換（互動 / 自動導覽）。
2. **階段二 · 學習互動**：點物件浮單字卡 / 例句 / 發音回饋 + 導師劇本對話框 + 選項按鈕 + 計分。
3. **階段三 · 視覺升級**：粒子、動態燈光、HUD 數據層、控制器 / 光線視覺、背景融入。
4. **階段四 · 選場景 / 選語言 + 打磨 + RWD + 觸控支援**（行動裝置用觸控拖曳）。

每階段獨立可驗證、可 commit。

## 不做（YAGNI）

- 不接真實 AI / LLM 對話。
- 不要求麥克風 / 語音辨識。
- 不新增後端或 serverless。
- 不做真正的 WebXR 頭盔模式（仍是螢幕上的 3D，標示為 VR 概念展示）。
- 不引入除 OrbitControls 外的重型 3D 函式庫。
